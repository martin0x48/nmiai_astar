# Session Report — 2026-03-22

## Objective

Improve the ML prediction pipeline for the Astar Island competition (NM i AI 2026). Starting baseline: XGB+LGBM mean ensemble (v3) with heuristic targeted smoothing, observing only seed 0.

## Changes made

### 1. Temperature scaling (replaces targeted smoothing)

**Problem:** The live API scores predictions against ~10 empirical samples drawn from the true distribution. Our model's raw predictions were overconfident — assigning too much probability mass to the most likely class. The old heuristic (smooth cells with max_prob > 0.90 toward uniform with alpha=0.05) was a crude fix.

**Solution:** Learned a single temperature parameter T via grid search on the N=10 proxy scorer across 18 rounds of leave-one-round-out cross-validation. Temperature scaling raises all probabilities to power 1/T and renormalizes: `p_new = p^(1/T) / Z`. T > 1 softens predictions.

**Result:** Optimal T = 1.09 (consistently across all folds). The old targeted smoothing was removed entirely.

**Proxy improvement: +0.46 points** (83.27 → 83.73). Wins on 15/18 rounds.

### 2. CatBoost as third ensemble member

**Problem:** The XGB+LGBM mean ensemble has limited diversity. Adding a third model with different regularization could reduce variance.

**Solution:** Added CatBoostRegressor (500 iterations, depth 6, lr 0.05) as a third member. Required `allow_const_label=True` and `bootstrap_type='No'` to handle output columns where some target classes have near-zero values for all dynamic cells.

**Result:** Small but consistent improvement.

**Proxy improvement: +0.08 on top of temperature** (83.73 → 83.81).

### 3. Avg5 observation strategy

**Problem:** We have 50 queries per round but only use 9 (all on seed 0). The remaining 41 queries are wasted.

**Idea from user:** Observe all 5 seeds, predict 5 times (once per observation's global features), average the output distributions. This ensembles over observation noise.

**Implementation:** 9 queries per seed × 5 seeds = 45 queries. Each observation yields different global features (settlement_ratio, empty_ratio, forest_ratio). For each target seed, the model runs 5 times with these 5 different global feature sets, and the 5 probability distributions are averaged before submission.

**Result:** Moderate improvement, concentrated on certain round types.

**Proxy improvement: +0.22 points** (84.15 → 84.37). Wins on 9/19 rounds.

**Important finding from the per-seed benchmark:** Using per-seed observations directly (each seed gets its own observation's globals, no averaging) was *worse* than seed-0-only by 0.48 points. Even oracle GT globals per seed were worse than seed-0-only. The explanation: a single noisy observation correlates with the noise in the live scorer's empirical target, which accidentally helps calibration. Averaging over multiple noisy observations preserves some of this effect while reducing variance.

### 4. N=10 live-like proxy scorer

Built a proper evaluation metric that approximates the live API's scoring:
- Sample N=10 values per cell from the ground truth distribution
- Build empirical frequency distribution
- Score prediction against this empirical target with entropy-weighted KL
- Average over 20 Monte Carlo draws

This proxy has MAE ~9.5 and correlation ~0.97 with historical live API scores, much better than the old offline metric which overestimated by a large margin.
